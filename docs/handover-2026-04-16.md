## 4. How the pipeline flows (quick reference)

### End-to-end backtest (Operation 10)

```
strategy-ready
     │
     ▼  run_backtest()
signal_precompute          — src/python/orchestrator/signal_precompute.py
     │                       rolls M1 → H1, computes entry+exit indicators,
     │                       forward-fills back to M1 (no lookahead)
     ▼
Rust forex_backtester.exe  — src/rust/crates/backtester
     │                       reads enriched Arrow, simulates trades,
     │                       writes trade-log.arrow, equity-curve.arrow,
     │                       metrics.arrow
     ▼
Result ingestion           — src/python/rust_bridge/result_ingester.py
     │                       writes SQLite rows, validates schemas
     ▼
Evidence pack              — src/python/analysis/{metrics_builder,narrative,
     │                       anomaly_detector,evidence_pack}.py
     ▼
review-pending  (operator gate: accept / reject / refine)
```

Typical run on 2025 EURUSD (~372k M1 bars): **22 seconds**.

### Optimization flow (Epic 5, all stories done)

```
backtest → review-pending ─(operator)─► optimizing
                                        │
                                        ├─ OptimizationOrchestrator
                                        │   (src/python/optimization/orchestrator.py)
                                        ├─ parse_strategy_params → ParameterSpace
                                        ├─ spawn OptimizationBatchDispatcher (or
                                        │   PersistentBatchDispatcher if enabled)
                                        ├─ for gen in 0..N:
                                        │     Rust batch_runner evaluates candidates
                                        │     FoldManager computes CV objective
                                        ├─ checkpoint every K generations
                                        └─ promote top candidates → promoted-candidates.arrow
                                        ▼
                                     validating (perturbation, walk-forward, CPCV, MC, regime)
                                        ▼
                                     scoring (DSR, confidence)
                                        ▼
                                     selection (operator gate)
```

Main e2e test: `tests/e2e/test_epic5_pipeline_proof.py` (71KB, 28 tests).
