## Recommended Changes
1. Replace the story’s session boundaries with the exact architecture schedule from `config/base.toml`; remove the conflicting London/NY/overlap defaults.
2. Add required provenance fields to the schema and manifest: `config_hash`, `input_hash`, `artifact_hash`, broker/account context, source references, and calibration window/sample size.
3. Change versioning semantics so a new version is created only when inputs change; unchanged inputs should return the same artifact or fail as a no-op.
4. Split `source` into `build_mode` and actual provenance metadata; do not let `research/tick_analysis/live_calibration` stand in for source attribution.
5. Add an operator-readable cost-model evidence artifact summarizing session values, data sources, sample sizes, and any research-derived assumptions.
6. Clarify the contract for `mean_spread_pips/std_spread/mean_slippage_pips/std_slippage`: define how the Rust consumer must use them in V1.
7. Add an approved-version pointer for cost models, analogous to `latest_confirmed_version`, so downstream stages never auto-consume unreviewed “latest” artifacts.
8. Remove the fallback that treats missing Story 2.2 research as acceptable for a pipeline-proof artifact, or mark it explicitly as a temporary fixture that cannot be promoted.
9. De-scope or split the automated analysis and broad CLI surface from the core schema/default-artifact work if the goal is a clean V1 proof.
10. Align cost-model persistence with the shared artifact infrastructure already emerging in the repo instead of introducing another local manifest/storage pattern.
