## Recommended Changes
1. Re-scope Story 3.4 to subprocess dispatch, structured error handling, progress polling, cancellation, and deterministic invocation only.
2. Remove manifest creation from Rust in this story and assign it back to Python’s artifact layer / Story 3.6.
3. Replace `crates/common/src/arrow_schemas.rs` as SSOT with `contracts/arrow_schemas.toml`; Rust and Python should validate against that contract.
4. Replace the ad hoc `checkpoint.json` design with the Story 3.2 checkpoint contract and reference the contract file directly.
5. Fix all architecture references in the story: D3, D14, and any “multi-crate workspace” citations.
6. Rewrite ACs 2, 6, 7, and 9 so they are externally verifiable and explicitly exclude ephemeral files from the reproducibility contract.
7. Clarify memory ownership: orchestrator decides admission/concurrency; Rust enforces its local budget once launched.
8. Move `ResultIngester` and any SQLite-facing behavior out of 3.4 unless it is strictly limited to validating that expected output files exist.
9. Add a short “published outputs vs runtime working files” section to prevent later FR18/FR61 contradictions.
10. Add a note that a minimal stub backtester is acceptable for 3.4, with real trade simulation and final output semantics delivered in 3.5/3.6.
